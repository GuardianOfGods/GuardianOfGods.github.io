Git và github là gì? Git đã cứu rỗi cuộc đời của developer như thế nào.

Created2022-09-29|Updated2022-10-01|ProgrammingGitBasic
|Post View:

Mở bài

Hello mọi người. Mình là Thụ.
Trước khi đến với bài viết về git thì mình sẽ kể cho các bạn nghe một câu chuyện chan chứa nỗi đau và nước mắt có liên quan về git. Các bạn hãy đọc và cảm nhận.

Vào một ngày đẹp trời như bao câu chuyện cổ tích khác, hồi còn là một sinh viên năm 2, mình đã có một ý tưởng sẽ làm một phần mềm lên danh sách các công việc (gọi tắt là To Do List). Sau một tuần dốc sức để hoàn thiện sản phẩm thì cuối cùng nó đã được hoàn thành xong. Nếu câu chuyện mà đến đây thì chắc hẳn nó đã có một cái kết có hậu. Nhưng đời không như là mơ, trong lúc đang dọn dẹp lại ngăn nắp các file trong thư mục thì mình lỡ tay đã xóa nhầm dự án. BÙM, công sức, mồ hôi, nước mắt đổ xuống sông xuống bể. Không có cái dại nào như cái dại này.

Mặc dù rất tức giận vì cái tay không biết suy nghĩ của mình nhưng mình vẫn phải điềm tĩnh. Mình cũng đã thử một vài cách như sử dụng một vài phần mềm scan khôi phục nhưng không hoạt động. Rồi mình đi tìm câu trả lời, trải qua muôn trùng vạn kiếp cũng là lúc mình gặp GIT. Một công cụ backup dữ liệu, một viên đá timestone, một cỗ máy thời gian của doreamon. Mặc dù vẫn không thể khôi phục được dữ liệu đã mất nhưng git đã giúp mình có thể trả giá cho mọi sai sót đã mắc phải về sau. Vậy Git là gì? Hãy cùng mình đi tìm hiểu nhé!

Git và github

Trước khi vào chủ đề này thì mình các bạn nên nắm một số khái niệm sau để dễ hiểu hơn.

  1. Repository: Giống như một folder. Đặc điểm khác là được hỗ trợ thêm một số tính năng và được quản lý bởi github
  2. Local - Remote: local là cách gọi cho máy tính, còn remote là cách gọi cho server.

Định nghĩa

Git là hệ thống mã nguồn mở quản lý hay phân tán dữ liệu của các phiên bản code và được sử dụng free . Đến đây chắc sẽ có nhiều bạn không hiểu định nghĩa ấy. Thực ra nó không hề khó hiểu như bạn nghĩ. Đầu tiên mỗi một chương trình máy tính khi được sản xuất ra sẽ được đánh số phiên bản giống như các mặt hàng tiêu dùng bên ngoài, cái này đơn giản dùng để quản lý sản phẩm. Từ đó git sinh ra nhằm mục đích lưu trữ và quản lý dữ liệu các phiên bản phần mềm của bạn. Cụ thể như mình có chia sẻ qua câu chuyện ở bên trên, git giúp chúng ta quay lại những thời điểm backup dữ liệu, có thể hiểu nôm na là khôi phục dữ liệu. Ngoài ra dùng git trong làm việc nhóm để có thể quản lý dữ liệu tốt hơn.

GitHub là một server hệ thống và quản lý dự án theo phiên bản code , nó giống như mạng xã hội cho lập trình viên. Một số tính năng cơ bản của github có thể kể đến như là xem commit, merge code, blame issue.

Cách thức hoạt động và câu lệnh git

Người dùng sẽ thông qua git để đẩy giữ liệu từ máy tính (local) lên github (remote) và ngược lại. Dữ liệu ở local sẽ được sao chép lên remote, từ đó đảm bảo việc quản lý và lưu trữ dữ liệu. Ở dưới đây mình sẽ liệt kê các câu lệnh cơ bản đủ để dùng trong những ngày đầu làm quen với git.

Nếu như coi remote và local là kho chứa dữ liệu thì thằng git chính là chiếc xe công ten nơ 18 bánh vận chuyển dữ liệu từ kho local sang kho remote. Vậy các thao tác của git được dùng như thế nào:
git init - Tạo ra thùng hàng (repository) ở local. Hãy sử dụng git init khi bạn đã có sẵn một folder dưới local.
git clone - Vận chuyển thùng hàng từ remote về local thông qua xe tải git. Hãy sử dụng git clone trong trường hợp bạn tạo repository trước.
git pull - Đồng bộ hàng (dữ liệu trong repository) trên kho remote về local.
git status - Kiểm tra xem hàng đã được chất lên xe chưa. Nếu hàng chưa được chất lên xe thì dòng lệnh sẽ báo đỏ, đã chất sẽ báo xanh.
git add . - Chất hết hàng lên xe.
git commit -m "message" - Đánh dấu hàng vận chuyển (thay đổi những gì, có những hàng hóa nào, … vv).
git push - Vận chuyện hàng từ local lên remote. Nếu hàng chưa được chất và được đánh dấu thì sẽ không push được lên.

Ảnh minh họa. Xin lỗi các bạn vì bức paint của mình :(

Kết

Đây chỉ bước đầu của git và github căn bản, ngoài ra nó còn cả một hệ sinh thái đằng trước đang chào đón bạn. Git nói chung khá rộng nên mình sẽ chia bài viết này thành nhiều bài viết. Chúc các bạn thành công trong việc làm quen với git. Cám ơn các bạn đã đọc.

Git funfact

Theo như mình tìm hiểu trên mạng thì cái tên của Git có khá nhiều nghĩa, trong đó có 2 cách hiểu phổ thông nhất cho tên gọi của Git:

  1. “Global information tracker”: Khi bạn đang có trạng thái tốt, công việc tiến triển thuận lợi.
  2. “Goddamn idiotic truckload of sh*t”: Khi việc của bạn gặp trục trặc
    Mã nguồn của Git được ghi là “trình quản lý thông tin từ địa ngục.”
